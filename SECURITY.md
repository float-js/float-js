# Security Policy

## Supported Versions

Security updates are provided for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 2.2.x   | Yes                |
| 2.1.x   | Yes                |
| 2.0.x   | Yes                |
| < 2.0   | No                 |

## Reporting a Vulnerability

To report security vulnerabilities:

1. Do not open public GitHub issues for security vulnerabilities
2. Email security details to the maintainers
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix if available

## Response Process

- Acknowledgment within 48 hours
- Initial assessment within 1 week
- Fix development and testing
- Coordinated disclosure after patch release

## Security Best Practices

When using Float.js:

- Keep dependencies updated
- Use environment variables for sensitive data
- Validate user inputs in API routes
- Follow secure coding practices
- Review dependencies regularly
- Enable security headers in production

## Disclosure Policy

- Security issues are fixed in private
- Patches are released as soon as ready
- CVE identifiers assigned when applicable
- Public disclosure after patch availability
